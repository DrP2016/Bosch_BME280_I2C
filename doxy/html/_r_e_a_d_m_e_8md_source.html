31 ### 5 - Print compensated BME280 ADC Data to Serial Output
32 ADC Values are compensated with formulas from official Bosch BME280 datasheet. Default calculation precision is 32Bit Integer, return values are Unsigned 32Bit Integer. Temperature returns a Signed 32Bit Integer. Pressure features calculation with 64Bit Integer precision, this returns an Unsigned 32Bit Integer, too. Double Precision for Calculation is also available for all ADC values, return datatype is then 'double'. Return values of 32 and 64 Bit precision functions do not carry a decimal point. You have to divide them by 100. Divide by 1000 for Humidity.

65 ### 6 - Optional Functions
66 This library offers extended functions to read the current Run State and write Oversampling Rates, StandBy Time, IIR Filter Coefficent and Mode to BME280.
67 
68 
69 The Sensor needs to be inited before using these functions!
70 
71 #### BME280 Run State - Reads the current Run State of the Sensor. See Table.
72 | Value (state) | Description |
73 |:---------------:|--------------:|
74 |-1 | Sensor not inited|
75 |0 | Sensor in Pause|
76 |1 | NVM data are being copied|
77 |2 | Conversion and Storing Results is running|
78 
79 You have can get the current Run State from the Sensor by running the following function:
80 ```c++
81 int8_t bme280state = BME280.state();
82 ```
83 #### BME280 T_SB Settings - Controls inactive duration 't_standby' in normal mode. See Table.
84 | Value (t_sb) | Time (ms) |
85 |:---------------:|--------------:|
86 |0b000 | 0.5|
87 |0b001 | 62.5|
88 |0b010 | 125|
89 |0b011 | 250|
90 |0b100 | 500|
91 |0b101 | 1000|
92 |0b110 | 10|
93 |0b111 | 20|
94 
95 #### BME280 FILTER Settings - Controls the time constant of the IIR filter. See Table.
96 | Value (filter) | Coefficent |
97 |:---------------:|--------------:|
98 |0b000 | OFF |
99 |0b001 | x2 |
100 |0b010 | x4 |
101 |0b011 | x8 |
102 |>0b100 | x16 |
103 
104 You have to set the StandBy Time for "Normal Mode" and Filter Coefficent before writing it to the BME280 with this function:
105 ```c++
106 BME280.filter_config( uint8_t t_sb, uint8_t filter );
107 ```
108 
109 You now can write these settings to the Sensor. This is done by using the following function:
110 ```c++
111 BME280.filter_write();
112 ```
113 
114 #### BME280 OSRS Settings - Controls the oversampling for measurements. See Table.
115 
116 | Value (osrs_x) | Rate |
117 |:---------------:|--------------:|
118 |0b000 | SKIPPED|
119 |0b001 | x1|
120 |0b010 | x2|
121 |0b011 | x4|
122 |0b100 | x8|
123 |>0b101 | x16|
124 
125 You have to set the Oversampling Rates for Temperature, Pressure and Humidity before writing it to the BME280 with this function:
126 ```c++
127 BME280.osrs_config(uint8_t osrs_p,uint8_t osrs_t,uint8_t osrs_h);
128 ```
129 You now can write these settings to the Sensor. This is done by using one of the following functions. Each of these Functions writes the Oversampling Rate AND the wished Sensor Mode:
130 ```c++
131 BME280.sleep(); // No readings are done, Registers are ok to read/write
132 BME280.forced(); // One reading is done, then the Sensor is in Sleep Mode again
133 BME280.normal(); // Continuous switching between reading and defined StandBy Time
134 ```
